AIメンバーは、チャットセッションであなたの代わりに作業するコーディングエージェントです。各メンバーには特定の役割、ワークスペース、能力があります。

## AIメンバーの追加

1. チャットセッションを開く
2. 右サイドバーで**AIメンバーを追加**をクリック
3. 必要な設定を入力

### 設定オプション

| フィールド | 必須 | 説明 |
|-------|----------|-------------|
| 名前 | はい | このエージェントの@メンションハンドル（例：`Coder`、`Writer`） |
| ベースコーディングエージェント | はい | 使用する基盤AIエージェント（ローカルにインストール必須） |
| システムプロンプト | はい | エージェントの役割と動作を定義する指示 |
| ワークスペースパス | はい | このエージェントが作業するディレクトリ |
| モデルバリアント | いいえ | 特定のモデル設定（利用可能な場合） |

<Note>
**名前**はセッション内で一意である必要があり、文字、数字、アンダースコア、ハイフンのみ含められます（最大10文字）。
</Note>

## サポートされているコーディングエージェント

ローカルにインストールされた任意のコーディングエージェントから選択：

- **Claude Code** - Anthropicのコーディングアシスタント
- **Codex** - OpenAIのコーディングエージェント
- **Gemini** - GoogleのAIアシスタント
- **Qwen Code** - Alibabaのコーディングモデル
- **Amp** - Sourcegraphのエージェント
- **Cursor Agent** - CursorのAIアシスタント
- **GitHub Copilot** - GitHubのコーディングコンパニオン
- その他...

<Tip>
システムにインストールされているエージェントのみが利用可能なオプションとして表示されます。
</Tip>

## 効果的なシステムプロンプトの作成

システムプロンプトはAIメンバーが何をするかを定義します。明確で焦点を絞ったものにしてください：

**良い例：**
- "あなたはドキュメントライターです。明確でユーザーフレンドリーなコンテンツに集中してください。"
- "あなたはコードレビュアーです。バグ、パフォーマンス問題、ベストプラクティスをチェックしてください。"
- "あなたはバックエンドエンジニアです。プロジェクトの既存パターンに従って機能を実装してください。"

**ヒント：**
- 役割と責任を明確に
- 制約や好みを言及
- 簡潔に - エージェントは明確な指示でより良く動作

## AIメンバーの管理

### メンバーの編集

サイドバーの既存メンバーをクリックして設定を変更します。

<Warning>
ワークスペースパスは作成後に変更できません。異なるワークスペースが必要な場合は、新しいAIメンバーを作成してください。
</Warning>

### メンバーの削除

1. サイドバーのメンバーをクリック
2. **削除**をクリック
3. 削除を確認

## メンバーの状態

各AIメンバーにはステータスインジケーターが表示されます：

| 状態 | 意味 |
|-------|---------|
| アイドル | タスク受け入れ準備完了 |
| 実行中 | 現在タスクを処理中 |
| 完了 | 最後のタスクが終了 |
| 失敗 | エラーが発生 |
