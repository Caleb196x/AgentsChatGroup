AIメンバーを設定したら、シンプルなメッセージングでタスクを割り当て、作業を調整できます。

## @メンションでタスクを割り当て

`@`に続けてメンバー名を使用してタスクを割り当て：

```
@Coder メールバリデーション付きのログインフォームを実装してください。
```

1つのメッセージで複数のエージェントをメンションすることもできます：

```
@Coder APIエンドポイントを実装
@Writer READMEに新しいエンドポイントをドキュメント化
```

### メンションセレクターの使用

便利なため、**エージェントをメンション**セレクターを使用：

1. メッセージ入力の**@**ボタンをクリック
2. 1つ以上のAIメンバーを選択
3. メッセージを作成
4. 選択したメンションが自動的に含まれる

<Tip>
両方の方法を組み合わせ可能 - メッセージに`@name`を入力し、セレクターで追加のメンションを追加。
</Tip>

## メッセージ機能

### テキストと画像

メッセージは以下をサポート：
- プレーンテキストの指示
- 画像添付（視覚的コンテキスト用）

### 以前のメッセージを参照

以前のメッセージを引用してコンテキストを提供：

1. 任意のメッセージにホバー
2. **返信**ボタンをクリック
3. 新しいメッセージが引用されたコンテンツを参照

これはフォローアップの指示や修正に便利です。

## タスク実行の仕組み

AIメンバーをメンションすると：

1. **受信** - エージェントがメッセージを確認
2. **実行中** - エージェントがタスクの作業を開始
3. **完了** - タスクが正常に終了
4. **失敗** - エージェントがエラーに遭遇

<Note>
各メンションの現在のステータスをメッセージ内で直接確認できます。
</Note>

### キュー動作

エージェントがすでに実行中の場合：
- 新しいタスクは自動的にキューに入る
- エージェントが利用可能になるとタスクは順番に実行
- ビジーなエージェントにタスクをキューイングすると確認が表示

### チェーンコラボレーション

エージェントは応答で他のエージェントをメンションでき、以下のようなワークフローが可能：

1. `@Engineer`にソリューション設計を依頼
2. Engineerが計画を作成し、`@Coder`に実装を依頼
3. Coderが実装し、`@Reviewer`にコードチェックを依頼

<Warning>
チェーントリガーには無限ループを防ぐ深度制限があります。エージェントは自分自身をメンションできません。
</Warning>

## エージェントの停止

実行中のエージェントを中断する必要がある場合：

1. サイドバーで実行中のメンバーを見つける
2. **停止**をクリック
3. エージェントが現在のタスクを停止

## ベストプラクティス

### 明確な指示

何が欲しいか具体的に：

**良い：**
```
@Coder ヘッダーにログアウトボタンを追加。セッションをクリアして/loginにリダイレクトすべき。
```

**曖昧：**
```
@Coder 認証を修正
```

### メンションごとに1つのタスク

複雑な作業は分割：

```
@Coder まず、ユーザープロファイルのデータベーススキーマを作成。
```

その後フォローアップ：

```
@Coder 次に、プロファイルCRUD操作のAPIエンドポイントを追加。
```

### 適切なエージェントを使用

タスクをエージェントの役割に合わせる：

- ドキュメントタスク → Writerエージェント
- コード実装 → Coderエージェント
- アーキテクチャの質問 → Engineerエージェント
