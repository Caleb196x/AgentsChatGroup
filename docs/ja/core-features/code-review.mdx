AIメンバーがコーディングタスクを完了すると、組み込みツールで作業をレビューし、セッション全体を管理できます。

## コード変更の表示

エージェントがタスクを完了すると、メッセージの下に**コード変更**カードが表示されます。

クリックして**差分ビューア**を開き、以下を表示：
- 追加、変更、削除されたファイル
- シンタックスハイライト付きの行ごとの変更
- 変更前後の比較

<Tip>
差分ビューアはgitで追跡された変更のみを表示します。ワークスペースがgitリポジトリであることを確認してください。
</Tip>

## ワークスペースドロワー

AIメンバーのワークスペースをクリックして**ワークスペースドロワー**を開き、以下を提供：

### ワークスペースパス
このエージェントが作業するディレクトリ。

### 実行履歴
このエージェントのすべてのタスク実行のリスト、表示内容：
- 各タスクの実行時刻
- 成功または失敗のステータス
- 出力への素早いアクセス

### 実行ログ
各実行の詳細ログ、デバッグやエージェントの動作理解に有用。

## 実行出力の理解

エージェントが実行するたびに複数の出力が生成されます：

| ファイル | 目的 |
|------|---------|
| 入力 | エージェントに送信されたタスクプロンプト |
| 出力 | エージェントの応答と要約 |
| ログ | 詳細な実行ログ |
| 差分 | コード変更（git変更がある場合） |

<Note>
4KBを超える大きな差分はパフォーマンスのため切り詰められます。完全な変更はgitクライアントで表示できます。
</Note>

## 新しいファイル

エージェントがgitで追跡されていない新しいファイルを作成すると、別の**未追跡ファイル**セクションに表示されます。

<Warning>
1MBを超える未追跡ファイルはキャプチャされません。
</Warning>

## 実行制御

### エージェントの停止

エージェントに時間がかかりすぎる場合や方向が間違っている場合：

1. サイドバーで実行中のエージェントを見つける（「実行中」ステータス表示）
2. **停止**ボタンをクリック
3. エージェントが現在のタスクを停止

<Tip>
エージェントを停止した後、より明確な指示で新しいメッセージを送信できます。
</Tip>

### タスクキュー

すでに実行中のエージェントにタスクを送信すると：
- 新しいタスクが自動的にキューに入る
- 確認ダイアログが表示
- エージェントが終了するとタスクが順番に実行

## コンテキスト管理

システムがAIエージェントの会話コンテキストを自動的に管理します。

### コンテキストの仕組み

各AIメンバーが受け取るもの：
- **セッションメンバー** - チャットに誰がいてその役割
- **最近のメッセージ** - 会話履歴
- **参照コンテンツ** - 引用されたメッセージや添付ファイル

### 自動コンテキスト圧縮

会話を効率的に保つため：
- 最大30メッセージがコンテキストに保持
- 最新5メッセージは常に完全に保持
- 古いメッセージは自動的に要約

<Note>
コンテキストを手動で管理する必要はありません - システムが重要な情報を保持しながら自動的に処理します。
</Note>

## セッション管理ツール

### メッセージ検索

検索機能を使用してセッション全体で特定のメッセージや議論を見つけます。

### 一括削除

クリーンアップモードで、複数のメッセージを一度に選択して削除できます。

### メッセージ履歴

完全な会話履歴が保持され、いつでも参照可能です。

### セッションのアーカイブ

プロジェクトが完了または一時停止した場合：

1. セッションの**アーカイブ**をクリック
2. セッションが読み取り専用になる
3. すべてのメッセージとメンバーが保持

後で作業を続けるには、**復元**をクリックしてセッションを再アクティブ化。

## 効果的なレビューのヒント

1. **まず差分をチェック** - どのコードが変更されたか素早く確認
2. **出力を読む** - エージェントが何をしようとしたか理解
3. **必要に応じてログを確認** - 何かおかしい場合は詳細を掘り下げ
4. **メンションでフォローアップ** - 必要に応じてエージェントに修正や説明を依頼

```
@Coder ログイン関数は良さそうですが、無効な認証情報のエラー処理を追加できますか？
```
