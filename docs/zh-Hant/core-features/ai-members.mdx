AI 成員是在聊天會話中代表您工作的程式設計代理。每個成員都有特定的角色、工作空間和能力。

## 新增 AI 成員

1. 開啟您的聊天會話
2. 在右側邊欄中，點擊**新增 AI 成員**
3. 填寫所需配置

### 配置選項

| 欄位 | 必填 | 描述 |
|-------|----------|-------------|
| 名稱 | 是 | 此代理的 @提及代號（如 `Coder`、`Writer`） |
| 基礎程式設計代理 | 是 | 要使用的底層 AI 代理（必須本地安裝） |
| 系統提示 | 是 | 定義代理角色和行為的指令 |
| 工作空間路徑 | 是 | 此代理工作的目錄 |
| 模型變體 | 否 | 特定模型配置（如可用） |

<Note>
**名稱**在會話中必須唯一，只能包含字母、數字、底線和連字號（最多 10 個字元）。
</Note>

## 支援的程式設計代理

從任何本地安裝的程式設計代理中選擇：

- **Claude Code** - Anthropic 的程式設計助手
- **Codex** - OpenAI 的程式設計代理
- **Gemini** - Google 的 AI 助手
- **Qwen Code** - 阿里巴巴的程式設計模型
- **Amp** - Sourcegraph 的代理
- **Cursor Agent** - Cursor 的 AI 助手
- **GitHub Copilot** - GitHub 的程式設計夥伴
- 還有更多...

<Tip>
只有系統上已安裝的代理才會顯示為可用選項。
</Tip>

## 編寫有效的系統提示

系統提示定義了您的 AI 成員做什麼。保持清晰和專注：

**好的範例：**
- "你是一名文件寫手。專注於清晰、使用者友善的內容。"
- "你是一名程式碼審查員。檢查錯誤、效能問題和最佳實踐。"
- "你是一名後端工程師。按照專案現有模式實作功能。"

**技巧：**
- 明確角色和職責
- 提及任何約束或偏好
- 保持簡潔 - 代理在清晰指導下工作效果更好

## 管理 AI 成員

### 編輯成員

點擊側邊欄中的現有成員以修改其配置。

<Warning>
工作空間路徑建立後無法變更。如果需要不同的工作空間，請建立新的 AI 成員。
</Warning>

### 刪除成員

1. 點擊側邊欄中的成員
2. 點擊**刪除**
3. 確認刪除

## 成員狀態

每個 AI 成員都會顯示狀態指示器：

| 狀態 | 含義 |
|-------|---------|
| 閒置 | 準備接收任務 |
| 執行中 | 當前正在處理任務 |
| 已完成 | 已完成上一個任務 |
| 失敗 | 遇到錯誤 |
