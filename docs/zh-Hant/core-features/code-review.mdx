當 AI 成員完成程式設計任務時，您可以透過內建工具審查他們的工作並管理整個會話。

## 檢視程式碼變更

代理完成任務後，您會在其訊息下方看到**程式碼變更**卡片。

點擊開啟**差異檢視器**，顯示：
- 新增、修改或刪除的檔案
- 帶語法高亮的逐行變更
- 前後對比

<Tip>
差異檢視器只顯示已追蹤的 git 變更。確保您的工作空間是 git 儲存庫。
</Tip>

## 工作空間抽屜

點擊 AI 成員的工作空間開啟**工作空間抽屜**，提供：

### 工作空間路徑
此代理工作的目錄。

### 執行歷史
此代理所有任務執行的列表，顯示：
- 每個任務的執行時間
- 成功或失敗狀態
- 快速存取輸出

### 執行日誌
每次執行的詳細日誌，用於除錯或了解代理做了什麼。

## 理解執行輸出

代理每次執行都會產生幾個輸出：

| 檔案 | 用途 |
|------|---------|
| 輸入 | 傳送給代理的任務提示 |
| 輸出 | 代理的回應和摘要 |
| 日誌 | 詳細的執行日誌 |
| 差異 | 程式碼變更（如果有 git 變更） |

<Note>
大於 4KB 的差異會被截斷以提高效能。您可以在 git 客戶端中檢視完整變更。
</Note>

## 新檔案

當代理建立的新檔案未被 git 追蹤時，它們會出現在單獨的**未追蹤檔案**部分。

<Warning>
大於 1MB 的未追蹤檔案不會被擷取。
</Warning>

## 執行控制

### 停止代理

如果代理耗時過長或方向錯誤：

1. 在側邊欄找到執行中的代理（顯示「執行中」狀態）
2. 點擊**停止**按鈕
3. 代理將停止目前任務

<Tip>
停止代理後，您可以傳送新訊息並提供更清晰的指令。
</Tip>

### 任務佇列

當您向正在執行的代理傳送任務時：
- 新任務自動排隊
- 出現確認對話方塊
- 代理完成後按順序執行任務

## 上下文管理

系統自動為您的 AI 代理管理對話上下文。

### 上下文如何運作

每個 AI 成員接收：
- **會話成員** - 聊天中有誰及其角色
- **最近訊息** - 對話歷史
- **引用內容** - 任何被引用的訊息或附件

### 自動上下文壓縮

為保持對話高效：
- 最多保留 30 則訊息在上下文中
- 最近 5 則訊息始終完整保留
- 舊訊息自動摘要

<Note>
您無需手動管理上下文 - 系統會自動處理，同時保留重要資訊。
</Note>

## 會話管理工具

### 搜尋訊息

使用搜尋功能在會話中尋找特定訊息或討論。

### 批次刪除

在清理模式下，您可以一次選擇並刪除多則訊息。

### 訊息歷史

完整的對話歷史被保留，隨時可供參考。

### 封存會話

當專案完成或暫停時：

1. 點擊會話上的**封存**
2. 會話變為唯讀
3. 所有訊息和成員被保留

要稍後繼續工作，點擊**還原**重新啟用會話。

## 有效審查的技巧

1. **首先檢查差異** - 快速查看程式碼有什麼變更
2. **閱讀輸出** - 了解代理打算做什麼
3. **需要時檢視日誌** - 當有問題時深入細節
4. **用提及跟進** - 如有需要，請代理修改或解釋

```
@Coder 登入函式看起來不錯，但你能為無效憑證新增錯誤處理嗎？
```
