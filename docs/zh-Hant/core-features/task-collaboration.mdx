設定好 AI 成員後，您可以透過簡單的訊息來分配任務並協調他們的工作。

## 使用 @提及分配任務

使用 `@` 加成員名稱來分配任務：

```
@Coder 請實作帶有電子郵件驗證的登入表單。
```

您也可以在一則訊息中提及多個代理：

```
@Coder 實作 API 端點
@Writer 在 README 中記錄新端點
```

### 使用提及選擇器

為方便起見，使用**提及代理**選擇器：

1. 點擊訊息輸入框中的 **@** 按鈕
2. 選擇一個或多個 AI 成員
3. 編寫您的訊息
4. 選取的提及會自動包含在內

<Tip>
您可以同時使用兩種方法 - 在訊息中輸入 `@name` 並使用選擇器新增額外的提及。
</Tip>

## 訊息功能

### 文字和圖片

訊息支援：
- 純文字指令
- 圖片附件（用於視覺上下文）

### 引用之前的訊息

引用之前的訊息以提供上下文：

1. 將滑鼠懸停在任何訊息上
2. 點擊**回覆**按鈕
3. 您的新訊息將引用被引用的內容

這對於後續指令或更正很有用。

## 任務執行流程

當您提及 AI 成員時：

1. **已接收** - 代理確認您的訊息
2. **執行中** - 代理開始處理任務
3. **已完成** - 任務成功完成
4. **失敗** - 代理遇到錯誤

<Note>
您可以直接在訊息中看到每個提及的目前狀態。
</Note>

### 佇列行為

如果代理已在執行：
- 新任務會自動排隊
- 代理閒置時任務按順序執行
- 向忙碌代理排隊任務時會看到確認

### 鏈式協作

代理可以在回覆中提及其他代理，實現如下工作流程：

1. 您請求 `@Engineer` 進行解決方案設計
2. Engineer 建立計畫並請求 `@Coder` 實作
3. Coder 實作後請求 `@Reviewer` 檢查程式碼

<Warning>
鏈式觸發有深度限制以防止無限迴圈。代理無法提及自己。
</Warning>

## 停止代理

如果需要中斷正在執行的代理：

1. 在側邊欄找到執行中的成員
2. 點擊**停止**
3. 代理將停止目前任務

## 最佳實踐

### 清晰的指令

明確說明您想要什麼：

**好的：**
```
@Coder 在頁首新增登出按鈕。它應該清除會話並重新導向到 /login。
```

**模糊的：**
```
@Coder 修復認證
```

### 每次提及一個任務

對於複雜工作，分解任務：

```
@Coder 首先，為使用者資料建立資料庫綱要。
```

然後跟進：

```
@Coder 現在新增資料 CRUD 操作的 API 端點。
```

### 使用合適的代理

將任務與代理角色匹配：

- 文件任務 → Writer 代理
- 程式碼實作 → Coder 代理
- 架構問題 → Engineer 代理
