AI 团队预设让您保存并复用 AI 成员配置，只需几秒钟即可启动完整的 AI 团队，而不必每次都从头配置每个代理。

预设分为两种类型：

- **成员预设** — 单个 AI 代理的配置（角色、系统提示、代理类型）
- **团队预设** — 成员预设的组合，可一次性全部导入到会话中

## 内置预设

AgentsChatGroup 内置了 **11 个成员预设**和 **5 个团队预设**，帮助您立即上手。

### 内置成员预设

| 预设 | 角色 |
|------|------|
| 架构师 | 系统设计与技术规划 |
| 后端工程师 | 服务端功能实现 |
| 前端工程师 | UI 与客户端开发 |
| 全栈工程师 | 端到端功能开发 |
| 代码审查员 | 安全、性能与最佳实践审查 |
| QA 测试员 | 测试计划与执行 |
| DevOps 工程师 | CI/CD、基础设施与部署 |
| 数据工程师 | 数据管道与 ETL |
| 产品经理 | 需求、路线图与优先级管理 |
| 技术写作 | 文档与用户指南 |
| 安全分析师 | 漏洞评估与安全加固 |

### 内置团队预设

| 团队 | 包含成员 |
|------|---------|
| 全栈开发团队 | 架构师 + 后端工程师 + 前端工程师 + 代码审查员 |
| 内容生产团队 | 产品经理 + 技术写作 + QA 测试员 |
| DevOps 发布团队 | 后端工程师 + DevOps 工程师 + 代码审查员 |
| 安全审计团队 | 安全分析师 + 后端工程师 + 代码审查员 |
| 数据平台团队 | 数据工程师 + 后端工程师 + QA 测试员 |

<Note>
内置预设无法删除。您可以禁用它们以从快速添加列表中隐藏，或者复制它们来创建自定义版本。
</Note>

## 管理预设

打开**设置 → 预设**来管理所有成员和团队模板。

### 成员预设标签页

- **添加**新的自定义成员预设，设置名称、代理类型和系统提示
- **编辑**现有的自定义预设
- **复制**任意预设（内置或自定义）作为起点
- **禁用**预设以在快速添加侧栏中隐藏它，而不删除
- **删除**不再需要的自定义预设

### 团队预设标签页

- **添加**新团队，选择成员预设并命名
- **编辑**自定义团队中的成员
- **复制**任意团队预设以创建变体
- **禁用**团队以从导入列表中隐藏
- **删除**自定义团队

## 在会话中使用预设

### 添加成员预设

在聊天右侧的 **AI 成员侧栏**中：

1. 点击成员列表下方的**从预设添加**
2. 浏览可用的成员预设
3. 点击预设，立即将该 AI 成员加入会话

新成员将以预设的名称、代理类型和系统提示创建。您仍然可以在添加后编辑成员来调整工作空间路径或其他字段。

### 导入团队预设

1. 在 AI 成员侧栏中点击**导入团队**
2. 选择要导入的团队预设
3. 查看**导入预览**——它会显示每个成员及其状态：
   - **创建** — 将添加新成员
   - **复用** — 会话中已有同名同配置的成员
   - **自动重命名** — 名称与现有成员冲突但配置不同；将自动添加 `_2` 后缀（或更高）
4. 确认导入

<Tip>
预览步骤让您在导入前发现命名冲突。如果会话中已有"Coder"，且导入的 Coder 预设配置不同，它将以"Coder_2"的名称导入。
</Tip>

### 冲突解决规则

导入团队时，AgentsChatGroup 自动应用以下规则：

| 场景 | 处理方式 |
|------|---------|
| 同名 + 同配置 | 复用已有成员——不创建重复 |
| 同名 + 不同配置 | 创建新成员，自动添加 `_2`（或 `_3` 等）后缀 |
| 成员已在当前会话中 | 跳过——不修改现有成员 |

## 归档会话

在**归档会话**中，导入控件为**禁用**状态。无法向归档会话添加成员或导入团队。
