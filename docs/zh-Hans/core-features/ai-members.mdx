AI 成员是在聊天会话中代表您工作的编程代理。每个成员都有特定的角色、工作空间和能力。

## 添加 AI 成员

1. 打开您的聊天会话
2. 在右侧边栏中，点击**添加 AI 成员**
3. 填写所需配置

### 配置选项

| 字段 | 必填 | 描述 |
|-------|----------|-------------|
| 名称 | 是 | 此代理的 @提及句柄（如 `Coder`、`Writer`） |
| 基础编程代理 | 是 | 要使用的底层 AI 代理（必须本地安装） |
| 系统提示 | 是 | 定义代理角色和行为的指令 |
| 工作空间路径 | 是 | 此代理工作的目录 |
| 模型变体 | 否 | 特定模型配置（如可用） |

<Note>
**名称**在会话中必须唯一，只能包含字母、数字、下划线和连字符（最多 10 个字符）。
</Note>

## 支持的编程代理

从任何本地安装的编程代理中选择：

- **Claude Code** - Anthropic 的编程助手
- **Codex** - OpenAI 的编程代理
- **Gemini** - Google 的 AI 助手
- **Qwen Code** - 阿里巴巴的编程模型
- **Amp** - Sourcegraph 的代理
- **Cursor Agent** - Cursor 的 AI 助手
- **GitHub Copilot** - GitHub 的编程伴侣
- 还有更多...

<Tip>
只有系统上已安装的代理才会显示为可用选项。
</Tip>

## 编写有效的系统提示

系统提示定义了您的 AI 成员做什么。保持清晰和专注：

**好的示例：**
- "你是一名文档写手。专注于清晰、用户友好的内容。"
- "你是一名代码审查员。检查错误、性能问题和最佳实践。"
- "你是一名后端工程师。按照项目现有模式实现功能。"

**技巧：**
- 明确角色和职责
- 提及任何约束或偏好
- 保持简洁 - 代理在清晰指导下工作效果更好

## 管理 AI 成员

### 编辑成员

点击侧边栏中的现有成员以修改其配置。

<Warning>
工作空间路径创建后无法更改。如果需要不同的工作空间，请创建新的 AI 成员。
</Warning>

### 删除成员

1. 点击侧边栏中的成员
2. 点击**删除**
3. 确认删除

## 成员状态

每个 AI 成员都会显示状态指示器：

| 状态 | 含义 |
|-------|---------|
| 空闲 | 准备接收任务 |
| 运行中 | 当前正在处理任务 |
| 已完成 | 已完成上一个任务 |
| 失败 | 遇到错误 |

## 从预设快速添加

无需从头填写所有字段，您可以使用**成员预设**——预配置好角色、代理类型和系统提示的模板——瞬间添加成员。

在 AI 成员侧栏中点击**从预设添加**，浏览可用模板。您还可以通过**导入团队预设**，一键将多个成员同时加入会话。
