设置好 AI 成员后，您可以通过简单的消息来分配任务并协调他们的工作。

## 使用 @提及分配任务

使用 `@` 加成员名称来分配任务：

```
@Coder 请实现带有邮箱验证的登录表单。
```

您也可以在一条消息中提及多个代理：

```
@Coder 实现 API 端点
@Writer 在 README 中记录新端点
```

### 使用提及选择器

为方便起见，使用**提及代理**选择器：

1. 点击消息输入框中的 **@** 按钮
2. 选择一个或多个 AI 成员
3. 编写您的消息
4. 选中的提及会自动包含在内

<Tip>
您可以同时使用两种方法 - 在消息中输入 `@name` 并使用选择器添加额外的提及。
</Tip>

## 消息功能

### 文本和图片

消息支持：
- 纯文本指令
- 图片附件（用于视觉上下文）

### 引用之前的消息

引用之前的消息以提供上下文：

1. 将鼠标悬停在任何消息上
2. 点击**回复**按钮
3. 您的新消息将引用被引用的内容

这对于后续指令或更正很有用。

## 任务执行流程

当您提及 AI 成员时：

1. **已接收** - 代理确认您的消息
2. **运行中** - 代理开始处理任务
3. **已完成** - 任务成功完成
4. **失败** - 代理遇到错误

<Note>
您可以直接在消息中看到每个提及的当前状态。
</Note>

### 队列行为

如果代理已在运行：
- 新任务会自动排队
- 代理空闲时任务按顺序执行
- 向繁忙代理排队任务时新任务会被忽略

### 链式协作

代理可以在回复中提及其他代理，实现如下的工作流：

1. 您请求 `@Engineer` 进行解决方案设计
2. Engineer 创建计划并请求 `@Coder` 实现
3. Coder 实现后请求 `@Reviewer` 检查代码

<Warning>
链式触发有深度限制以防止无限循环。代理无法提及自己。
</Warning>

## 停止代理

如果需要中断正在运行的代理：

1. 在侧边栏找到运行中的成员
2. 点击**停止**
3. 代理将停止当前任务

## 最佳实践

### 清晰的指令

明确说明您想要什么：

**好的：**
```
@Coder 在页头添加登出按钮。它应该清除会话并重定向到 /login。
```

**模糊的：**
```
@Coder 修复认证
```

### 分解复杂任务

对于复杂工作，可以设置一个监管者分解任务并分配给不同的agent：

```
@Designer 目标是为用户创建数据库。
先将任务按能力拆分、分配到不同agent，每个任务完成后让不同agent通知reviewer进行评估。
评估不通过的话，将评审意见反馈给相关agent进行修改，直到评审通过。
```

### 使用合适的代理

将不同类型的任务与交给擅长的代理Agent完成，例如（仅供参考）：

- 文档任务 → Codex 代理
- 代码实现 → Claude Code 代理
- 架构问题 → Codex 代理
- UI设计 → Gemini CLI 代理
