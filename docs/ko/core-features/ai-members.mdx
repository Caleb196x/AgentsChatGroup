AI 멤버는 채팅 세션에서 여러분을 대신해 작업하는 코딩 에이전트입니다. 각 멤버는 특정 역할, 작업 공간, 기능을 가집니다.

## AI 멤버 추가

1. 채팅 세션 열기
2. 오른쪽 사이드바에서 **AI 멤버 추가** 클릭
3. 필요한 구성 입력

### 구성 옵션

| 필드 | 필수 | 설명 |
|-------|----------|-------------|
| 이름 | 예 | 이 에이전트의 @멘션 핸들 (예: `Coder`, `Writer`) |
| 기본 코딩 에이전트 | 예 | 사용할 기본 AI 에이전트 (로컬 설치 필수) |
| 시스템 프롬프트 | 예 | 에이전트의 역할과 동작을 정의하는 지침 |
| 작업 공간 경로 | 예 | 이 에이전트가 작업할 디렉토리 |
| 모델 변형 | 아니오 | 특정 모델 구성 (사용 가능한 경우) |

<Note>
**이름**은 세션 내에서 고유해야 하며, 문자, 숫자, 밑줄, 하이픈만 포함할 수 있습니다 (최대 10자).
</Note>

## 지원되는 코딩 에이전트

로컬에 설치된 코딩 에이전트 중에서 선택:

- **Claude Code** - Anthropic의 코딩 어시스턴트
- **Codex** - OpenAI의 코딩 에이전트
- **Gemini** - Google의 AI 어시스턴트
- **Qwen Code** - Alibaba의 코딩 모델
- **Amp** - Sourcegraph의 에이전트
- **Cursor Agent** - Cursor의 AI 어시스턴트
- **GitHub Copilot** - GitHub의 코딩 컴패니언
- 그 외 다수...

<Tip>
시스템에 설치된 에이전트만 사용 가능한 옵션으로 표시됩니다.
</Tip>

## 효과적인 시스템 프롬프트 작성

시스템 프롬프트는 AI 멤버가 무엇을 하는지 정의합니다. 명확하고 집중적으로 유지하세요:

**좋은 예:**
- "당신은 문서 작성자입니다. 명확하고 사용자 친화적인 콘텐츠에 집중하세요."
- "당신은 코드 리뷰어입니다. 버그, 성능 문제, 모범 사례를 확인하세요."
- "당신은 백엔드 엔지니어입니다. 프로젝트의 기존 패턴을 따라 기능을 구현하세요."

**팁:**
- 역할과 책임을 명확히
- 제약 조건이나 선호도 언급
- 간결하게 유지 - 에이전트는 명확한 지시에서 더 잘 작동

## AI 멤버 관리

### 멤버 편집

사이드바의 기존 멤버를 클릭하여 구성을 수정합니다.

<Warning>
작업 공간 경로는 생성 후 변경할 수 없습니다. 다른 작업 공간이 필요하면 새 AI 멤버를 만드세요.
</Warning>

### 멤버 삭제

1. 사이드바에서 멤버 클릭
2. **삭제** 클릭
3. 삭제 확인

## 멤버 상태

각 AI 멤버에 상태 표시기가 표시됩니다:

| 상태 | 의미 |
|-------|---------|
| 유휴 | 작업 수신 준비 완료 |
| 실행 중 | 현재 작업 처리 중 |
| 완료 | 마지막 작업 완료 |
| 실패 | 오류 발생 |
