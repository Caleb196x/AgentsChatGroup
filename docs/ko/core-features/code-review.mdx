AI 멤버가 코딩 작업을 완료하면 내장 도구를 통해 작업을 리뷰하고 전체 세션을 관리할 수 있습니다.

## 코드 변경 보기

에이전트가 작업을 완료하면 메시지 아래에 **코드 변경** 카드가 표시됩니다.

클릭하여 **Diff 뷰어**를 열고 다음을 표시:
- 추가, 수정 또는 삭제된 파일
- 구문 강조가 있는 줄별 변경
- 전후 비교

<Tip>
Diff 뷰어는 git으로 추적된 변경만 표시합니다. 작업 공간이 git 저장소인지 확인하세요.
</Tip>

## 작업 공간 드로어

AI 멤버의 작업 공간을 클릭하여 **작업 공간 드로어**를 열고 다음을 제공:

### 작업 공간 경로
이 에이전트가 작업하는 디렉토리.

### 실행 기록
이 에이전트의 모든 작업 실행 목록, 표시 내용:
- 각 작업 실행 시간
- 성공 또는 실패 상태
- 출력에 빠른 접근

### 실행 로그
각 실행의 상세 로그, 디버깅이나 에이전트 동작 이해에 유용.

## 실행 출력 이해

에이전트가 실행될 때마다 여러 출력 생성:

| 파일 | 목적 |
|------|---------|
| 입력 | 에이전트에 전송된 작업 프롬프트 |
| 출력 | 에이전트의 응답과 요약 |
| 로그 | 상세 실행 로그 |
| Diff | 코드 변경 (git 변경이 있는 경우) |

<Note>
4KB를 초과하는 큰 diff는 성능을 위해 잘립니다. git 클라이언트에서 전체 변경을 볼 수 있습니다.
</Note>

## 새 파일

에이전트가 git으로 추적되지 않는 새 파일을 만들면 별도의 **추적되지 않은 파일** 섹션에 표시됩니다.

<Warning>
1MB를 초과하는 추적되지 않은 파일은 캡처되지 않습니다.
</Warning>

## 실행 제어

### 에이전트 중지

에이전트가 너무 오래 걸리거나 방향이 잘못된 경우:

1. 사이드바에서 실행 중인 에이전트 찾기 ("실행 중" 상태 표시)
2. **중지** 버튼 클릭
3. 에이전트가 현재 작업을 중지

<Tip>
에이전트를 중지한 후 더 명확한 지침으로 새 메시지를 보낼 수 있습니다.
</Tip>

### 작업 대기열

이미 실행 중인 에이전트에게 작업을 보내면:
- 새 작업이 자동으로 대기열에 추가됨
- 확인 대화 상자가 나타남
- 에이전트가 완료되면 작업이 순서대로 실행

## 컨텍스트 관리

시스템이 AI 에이전트의 대화 컨텍스트를 자동으로 관리합니다.

### 컨텍스트 작동 방식

각 AI 멤버가 받는 것:
- **세션 멤버** - 채팅에 누가 있고 그들의 역할
- **최근 메시지** - 대화 기록
- **참조된 콘텐츠** - 인용된 메시지나 첨부 파일

### 자동 컨텍스트 압축

대화를 효율적으로 유지하기 위해:
- 최대 30개 메시지가 컨텍스트에 유지됨
- 가장 최근 5개 메시지는 항상 완전히 유지
- 오래된 메시지는 자동으로 요약

<Note>
컨텍스트를 수동으로 관리할 필요가 없습니다 - 시스템이 중요한 정보를 보존하면서 자동으로 처리합니다.
</Note>

## 세션 관리 도구

### 메시지 검색

검색 기능을 사용하여 세션 전체에서 특정 메시지나 토론을 찾습니다.

### 일괄 삭제

정리 모드에서 여러 메시지를 한 번에 선택하고 삭제할 수 있습니다.

### 메시지 기록

전체 대화 기록이 보존되어 언제든지 참조할 수 있습니다.

### 세션 아카이브

프로젝트가 완료되거나 일시 중지된 경우:

1. 세션의 **아카이브** 클릭
2. 세션이 읽기 전용이 됨
3. 모든 메시지와 멤버가 보존됨

나중에 작업을 계속하려면 **복원**을 클릭하여 세션을 다시 활성화.

## 효과적인 리뷰를 위한 팁

1. **먼저 diff 확인** - 어떤 코드가 변경되었는지 빠르게 확인
2. **출력 읽기** - 에이전트가 무엇을 하려 했는지 이해
3. **필요시 로그 확인** - 뭔가 이상해 보일 때 세부 사항 파악
4. **멘션으로 후속 조치** - 필요하면 에이전트에게 수정이나 설명 요청

```
@Coder 로그인 함수 좋아 보이는데, 잘못된 자격 증명에 대한 오류 처리를 추가해줄 수 있어?
```
