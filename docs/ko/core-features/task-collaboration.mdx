---

AI 멤버를 설정했으면 간단한 메시징을 통해 작업을 할당하고 조율할 수 있습니다.

## @멘션으로 작업 할당

`@` 다음에 멤버 이름을 사용하여 작업 할당:

```
@Coder 이메일 유효성 검사가 있는 로그인 폼을 구현해주세요.
```

한 메시지에서 여러 에이전트를 멘션할 수도 있습니다:

```
@Coder API 엔드포인트 구현
@Writer README에 새 엔드포인트 문서화
```

### 멘션 선택기 사용

편의를 위해 **에이전트 멘션** 선택기 사용:

1. 메시지 입력의 **@** 버튼 클릭
2. 하나 이상의 AI 멤버 선택
3. 메시지 작성
4. 선택한 멘션이 자동으로 포함됨

<Tip>
두 방법을 결합할 수 있습니다 - 메시지에 `@name`을 입력하고 선택기로 추가 멘션을 추가.
</Tip>

## 메시지 기능

### 텍스트와 이미지

메시지 지원:
- 일반 텍스트 지침
- 이미지 첨부 (시각적 컨텍스트용)

### 이전 메시지 참조

이전 메시지를 인용하여 컨텍스트 제공:

1. 메시지에 마우스 호버
2. **답장** 버튼 클릭
3. 새 메시지가 인용된 내용을 참조

이는 후속 지침이나 수정에 유용합니다.

## 작업 실행 방식

AI 멤버를 멘션하면:

1. **수신됨** - 에이전트가 메시지 확인
2. **실행 중** - 에이전트가 작업 시작
3. **완료** - 작업이 성공적으로 끝남
4. **실패** - 에이전트가 오류를 만남

<Note>
메시지에서 각 멘션의 현재 상태를 직접 볼 수 있습니다.
</Note>

### 대기열 동작

에이전트가 이미 실행 중인 경우:
- 새 작업이 자동으로 대기열에 추가됨
- 에이전트가 사용 가능해지면 작업이 순서대로 실행
- 바쁜 에이전트에 작업을 대기열에 추가하면 확인이 표시됨

### 체인 협업

에이전트는 응답에서 다른 에이전트를 멘션하여 다음과 같은 워크플로우 가능:

1. `@Engineer`에게 솔루션 설계 요청
2. Engineer가 계획을 만들고 `@Coder`에게 구현 요청
3. Coder가 구현하고 `@Reviewer`에게 코드 확인 요청

<Warning>
체인 트리거에는 무한 루프를 방지하기 위한 깊이 제한이 있습니다. 에이전트는 자신을 멘션할 수 없습니다.
</Warning>

## 에이전트 중지

실행 중인 에이전트를 중단해야 하는 경우:

1. 사이드바에서 실행 중인 멤버 찾기
2. **중지** 클릭
3. 에이전트가 현재 작업을 중지

## 모범 사례

### 명확한 지침

원하는 것을 구체적으로:

**좋음:**
```
@Coder 헤더에 로그아웃 버튼을 추가해. 세션을 지우고 /login으로 리디렉션해야 해.
```

**모호함:**
```
@Coder 인증 수정
```

### 멘션당 하나의 작업

복잡한 작업은 분해:

```
@Coder 먼저 사용자 프로필용 데이터베이스 스키마를 만들어.
```

그런 다음 후속:

```
@Coder 이제 프로필 CRUD 작업을 위한 API 엔드포인트를 추가해.
```

### 적절한 에이전트 사용

작업을 에이전트 역할에 맞추기:

- 문서 작업 → Writer 에이전트
- 코드 구현 → Coder 에이전트
- 아키텍처 질문 → Engineer 에이전트
