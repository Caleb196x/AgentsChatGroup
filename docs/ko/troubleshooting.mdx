## 에이전트가 코드베이스가 비어있다고 보고

새 작업을 생성할 때 코딩 에이전트가 코드베이스가 비어있다고 보고하면, 저장소에서 Git의 sparse-checkout 기능이 활성화되어 있을 수 있습니다.

**해결책:**

저장소 루트 디렉토리에서 다음 명령을 실행:

```bash
git sparse-checkout disable
```

sparse-checkout을 비활성화한 후 새 작업을 생성하고 다시 시도하세요.

## 디버그 로그 활성화

문제 디버깅에 도움이 되는 더 자세한 로그가 필요한 경우, `RUST_LOG` 환경 변수를 설정하여 디버그 레벨 로깅을 활성화할 수 있습니다.

**사용법:**

```bash
RUST_LOG=debug npx agents-chatgroup
```

이렇게 하면 문제의 근본 원인을 식별하는 데 도움이 되는 상세한 로깅 출력이 제공됩니다.

## 위험: 데이터베이스 초기화

복구할 수 없는 오류가 발생하여 agents-chatgroup 데이터베이스를 완전히 초기화해야 하는 경우, 운영 체제의 애플리케이션 데이터 디렉토리를 삭제할 수 있습니다.

<Warning>
이 작업은 되돌릴 수 없으며 모든 작업과 설정이 손실됩니다. 진행하기 전에 중요한 정보를 백업하세요.
</Warning>

**OS에 따라 다음 디렉토리를 삭제:**

<Tabs>
<Tab title="macOS">
```bash
~/Library/Application Support/ai.starterra.ai.agents-chatgroup/
```
</Tab>

<Tab title="Linux">
```bash
~/.local/share/agents-chatgroup/
```
</Tab>

<Tab title="Windows">
```
%APPDATA%\starterra.ai\agents-chatgroup\
```

일반적으로: `C:\Users\<사용자 이름>\AppData\Roaming\starterra.ai\agents-chatgroup\`
</Tab>
</Tabs>

애플리케이션 데이터 디렉토리를 삭제한 후 agents-chatgroup을 다시 시작하면 빈 데이터베이스와 기본 설정으로 재설정됩니다.
